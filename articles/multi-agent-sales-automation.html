纳杰的销售引擎：品牌端·销售端·收款端，三端一体


> 在知识产权专业服务里，"卖"从来不是一次成交就结束的事。它是一个从品牌信任建立到持续服务的无限循环。我们把这条链拆成了三段，用数字员工跑通。


先讲一个真实场景。


上周，一个做跨境电子的客户通过公众号找到我们。他说在网上看到了一篇关于"AI商标申请趋势"的文章，觉得写的跟他的处境对得上，想聊聊。


从这篇文章被他看到，到最后委託我们处理越南商标布局——中间发生的每一步，不是靠运气，而是靠一套跑了大半年的数字员工系统在推动。


这套系统的设计逻辑很直接：**销售不是一段流程，它是一个可以拆解、可以运营的系统。** 我们把它拆成三段——品牌端、销售端、收款端——三段各有人格和能力，三段之间靠数据流动串联。


---


品牌端：让对的人在对的时间看到你


品牌端是整个销售引擎的入口。它的目标不是"宣传品牌"，而是**持续地产出让潜在客户能找到我们的专业内容**。


品牌端的数字员工做三件事：


**一、每日行业扫描与方向判断。**


一个调研智能体每天独立运行，扫描国知局官网的公告公报、监测行业动态、分析同行选题。它不写文章，但它决定"明天应该写什么"——确保我们的内容永远踩在客户最新关心的话题上。


**二、持续的内容生产与全渠道分发。**


每天一篇公众号文章，从选题到完稿到配图到评分到发布，5个智能体接力完成。一篇文章产出后，一个专门的推广智能体把它改编成知乎回答体、微博短帖、B站脚本——自动分发到8个渠道。


**三、建立可搜索的专业资产库。**


每一篇文章不只是"发了就过了"。它是被搜索引擎收录的、被知乎推荐算法分发的、被同行转发的——长期地、被动地带来潜在客户的注意力。品牌端的信号是线索——不是合同，不是定金，只是"有人对我们产生了兴趣"。


---


销售端：独立闭环，自己的节奏


潜在客户产生兴趣后，进入销售端。


**销售端是一个完全独立的闭环。** 从客户第一次接触到最后委托意向确认，所有环节在闭环内完成，闭环外的人只需要做两件事：知情的决策和最终的握手。


销售端由五个智能体接力，每个有独立人格和专职：


**Profiler（客户画像师）**—— 线索进来第一站。它不签单、不报价、不催款，只做一件事：把模糊的客户信息变成标准化的六维画像（业务类型、来源、销售阶段、行业、能力评分、客户价值）。六个维度打标完整了，才能进入下一步。


**Strategist（作战参谋）**—— 拿到六维画像后，输出三步作战方案。它脑子里装着8 Step方法论（单一销售目标设定→有效约见理由→拜访执行→方案报价→谈判成交→维护服务→再成交），但不背书，只给当前客户最需要的两三个动作。产出是带着具体话术和行动承诺的作战方案。


**International（涉外专家）**—— 按需启动。当客户有涉外需求时，从 foreign_ip.db 中查目标国家的合作所（862条·61国）、调年费报价（958条）、查工作日日历（16790天），输出国别策略。一条 SQL 能找到「这个案子在菲律宾建议找谁、多少钱、多久」。


**Sgate（商务质检）**—— 管线的守门人。对画像和策略做 A-E 五维能力评分（独立开发/沟通见面/谈判成交/维护再成交/整合资源），15项检视中如果核心项不通过，整份打回重做。它不挑刺，它防止商务人员带着不成熟的东西去见客户。


**Trainer（晨会教练）**—— 每天自动生成一个15分钟训练科目，针对能力短板做垂直训练。今天练话术、明天练谈判、后天练约见理由冲击。训练素材来自真实成单案例复盘和 common-qa.md 话术库。


销售端的产出只有一个信号：签约。


---


收款端：贯穿全线，管钱也管帐


收款端是整个销售引擎里**最特殊的一段**——它不只在签完合同之后才干活，而是**从第一天起就贯穿全线运行。**


收款端有两个智能体并行工作：


**Receivable（应收款主管）**—— 从报价到结算的完整钱流闭环。它做五件事：

1. **报价预审**—— 签约前审核成本和折扣策略，双轨输出含税/不含税方案

2. **合同登记**—— 记录特殊条款和案件关联

3. **收款跟踪**—— 分期、余款、发票、退款全追踪

4. **结算分析**—— 增值税处理、官费抵扣、成本覆盖率

5. **提成计算**—— 按 CRM 公式算贡献额×提成比例


**年费管家（Annual Fee Manager）**—— 每天自动扫描2000多件专利的年费到期表，跟缴费记录交叉比对，标记即将到期、已到期未缴、缴费数据异常的案件，推送到相关人员。


这不是等到年底才做的事，不是按月做的事——它是**每天**在跑。不管客户在哪一端：新线索进来看他的存量专利谁在管；在谈的客户方案里有年费预估；已签客户的缴费状态实时更新。


**财务智能（Financial Intelligence）**—— 收款端同时承担着全套财务分析的职能。它关注的不是"谁付了多少钱"，而是几个更关键的问题：


**● 品牌端花了多少钱，带来了多少线索？**

公众号运营成本、内容生产投入、各渠道分发的人力与API费用——这些支出跟品牌端产生的线索量、线索质量做对比，算出获客成本。


**● 销售端的人力成本跟转化率匹配吗？**

商务人员的薪酬、招聘新人的投入、培训费用——这些固定支出跟销售端的签约率和客单价做对比，判断人效是否合理。


**● 活动和展会的投入产出比是多少？**

每次参加行业展会的展位费、差旅费、物料费——对比活动中获取的线索数量和后续转化，判断哪些展会值得继续参加。


**● 整体预算执行情况怎么样？**

每个月出一份预算执行简报：各端的实际支出vs预算、偏差原因、下月调整建议。


这些分析不是等到月底再拍脑袋，而是有数据支撑的、按周期的自动化输出。**品牌端和销售端在前线作战，收款端在后面算账——三端的健康度，都反映在收款端的报表里。**


---


三端的数据闭环


三端的数字员工不是各自为战的孤岛。它们的产出汇聚成一条完整的数据链：




品牌端（7 Agent）


expansion → research → writer → visual → gate → outreach → promotion


市场方向 → 调研笔记 → 文章 → 封面 → 评分 → 发布 → 推广


产出：线索


↓


销售端（5 Agent + 1 按需）


profiler → strategist → [international] → sgate → trainer


六维画像 → 作战方案 → 涉外策略 → 质检 → 晨训


产出：签约


↓


收款端（2 Agent + 财务智能）


receivable（报价→合同→收款→结算→提成）


annual-fee-manager（年费监控·贯穿全线）


产出：收入 + 分析


↑_____________ 数据回流 _____________↑






客户的委托完成了，交付过程中的新案例成为品牌端下一篇内容的素材。收款端在这个过程中持续记录每一笔支出和每一笔收入——它不只在终点线等着，而是全程在场。

最后说一句




这套系统的总成本——包含所有API调用、服务器、维护——月均不到200块。这个数字本身，也是收款端在算。




我们把这些架构和部分代码做了开源，放在了 `ipfancy.github.io/najie-articles` 上。目的不是炫技，而是想表达：




**纳杰和觅理做的每一件事，出发点都是用当下最好的技术手段，让合作伙伴的诉求得到最及时的响应和最靠谱的执行。**
